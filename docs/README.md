# Documentation

This directory contains project-level documentation, conceptual notes, and public-facing writing.

## Directory Structure

- `blog/`: Public-facing blog drafts and posts
- `weekly-notes-outputs/`: Document-style outputs derived from weekly study notes
- `project-overview.md`: Overall project description
- `README.md`: Documentation directory guide

## Purpose

The `docs` directory is used for materials that explain the project beyond weekly activity logs.

This includes:

- project overview
- conceptual summaries
- blog drafts
- public-facing explanations
- document-style outputs derived from weekly notes

## Difference from Other Directories

- `weekly-notes/`: session-by-session study and research records
- `reports/`: milestone-level reports
- `figures/`: figures and diagrams
- `results/`: numerical outputs and analysis summaries
- `scripts/`: executable analysis code
- `notebooks/`: exploratory coding and analysis notes

## Writing Principle

Documentation should be clear enough for another person to understand:

1. What the project is about
2. Why a method or concept matters
3. How each activity connects to the larger BCI robot-control pipeline
